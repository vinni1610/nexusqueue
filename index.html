Advanced Workflows

Orchestrate complex pipelines with ease

Build multi-step workflows for AI tasks, billing pipelines, media processing, and reporting systems that require reliable coordination across distributed jobs.

  • ✓
    Fully Distributed
    Workflows run across nodes, ensuring high availability and fault-tolerant execution at any scale.
  • ✓
    Cascading Context
    Pass cumulative context between jobs seamlessly. Build workflows from plain functions.
  • ✓
    Nested Sub-Workflows
    Compose complex workflows hierarchically for better organization and reusability.
lib/myapp/workflow.ex Elixir 
defmodule MyApp.Workflow do
  alias NexusQueue.Workflow

  def invoke(params) do
    Workflow.new()
    |> Workflow.put_context(params)
    |> Workflow.add_step(:fetch, &fetch_data/1)
    |> Workflow.add_step(:process,
         &process/1, deps: :fetch)
    |> Workflow.add_step(:notify,
         &notify/1, deps: :process)
    |> Workflow.run()
  end
end

Smart Concurrency

Fine-grained control across your cluster

NexusQueue Pro's Smart engine provides advanced concurrency control. With global limits, rate limiting, partitioning for granular control, and burst mode to maximize throughput.

  • ✓
    Granular Partitioning
    Split queues into partitions, each with independent concurrency and rate limits.
  • ✓
    Distributed Rate Limiting
    Control execution rates across your entire cluster within configurable time windows.
  • ✓
    Burst Performance
    Temporarily exceed per-partition limits when spare capacity is available in the system.
